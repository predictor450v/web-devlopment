What Are Events?

Definition

An event is an action or occurrence that happens in the browser — like a user clicking a button, pressing a key, submitting a form, or the page finishing loading. JavaScript can listen for these events and execute code in response.

How Events Work

  1. Something happens — user clicks, types, scrolls, etc.
  2. Browser creates an Event object — containing details about what happened.
  3. Event is dispatched — the browser sends it to the relevant element.
  4. Event listeners fire — any attached handler functions execute.

Three Ways to Add Event Listeners

Method 1 Not Recommended

Inline HTML Event Handler

Directly in the HTML attribute. Mixes HTML with JS — hard to maintain.

HTML inline 
<button onclick="alert('Clicked!')">Click Me</button>
Method 2 Okay, But Limited

DOM Property (on-event)

Assign a function to an element's event property. Only one handler per event can be attached this way.

JavaScript on-event 
const btn = document.getElementById('myBtn');
btn.onclick = function() {
    alert('Clicked!');
};
// This REPLACES the previous handler:
btn.onclick = function() {
    alert('Only this one runs!');
};
Method 3 Best Practice

addEventListener()

The modern, recommended way. Supports multiple handlers, options (capture, once, passive), and removeEventListener().

JavaScript addEventListener 
const btn = document.getElementById('myBtn');

// Multiple handlers — BOTH will run!
btn.addEventListener('click', function() {
    console.log('Handler 1');
});
btn.addEventListener('click', function() {
    console.log('Handler 2');
});

// Options object:
btn.addEventListener('click', handler, {
    capture: false,  // use capturing phase?
    once: true,      // auto-remove after first fire
    passive: true    // won't call preventDefault()
});
Bonus Removing Listeners

removeEventListener()

To remove an event listener, you must pass the same function reference. Anonymous functions cannot be removed!

JavaScript remove 
function handleClick() {
    console.log('Clicked!');
}

btn.addEventListener('click', handleClick);
btn.removeEventListener('click', handleClick); // Works

// This WON'T work — different function reference:
btn.addEventListener('click', () => console.log('Hi'));
btn.removeEventListener('click', () => console.log('Hi'));

Detecting Key Combinations

JavaScript combos 
document.addEventListener('keydown', (e) => {
    // Ctrl + S  (Save shortcut)
    if (e.ctrlKey && e.key === 's') {
        e.preventDefault(); // prevent browser save
        console.log('Custom save!');
    }

    // Ctrl + Shift + K
    if (e.ctrlKey && e.shiftKey && e.key === 'K') {
        console.log('Secret combo!');
    }

    // Escape key
    if (e.key === 'Escape') {
        closeModal();
    }
});

Form Submit Event

The submit event fires when a form is submitted. Use e.preventDefault() to stop the page from reloading.

JavaScript submit 
form.addEventListener('submit', (e) => {
    e.preventDefault();     // ← Stop page reload!
    const formData = new FormData(e.target);
    const name = formData.get('name');
    console.log('Submitted:', name);
});

The select event fires when text is selected. Clipboard events include copy, cut, and paste.

The Event Object

What is the Event Object?

When an event fires, the browser automatically passes an Event object to the handler function. This object contains all the details about the event.

JavaScript event object 
element.addEventListener('click', function(event) {
    // The 'event' parameter (often called 'e') is the Event object
    console.log(event);
});

Key Properties & Methods

Property / Method Description
e.typeThe type of event ("click", "keydown", etc.)
e.targetThe element that triggered the event
e.currentTargetThe element the listener is attached to
e.timeStampWhen the event occurred (ms since page load)
e.preventDefault()Stops the default browser action
e.stopPropagation()Stops event from bubbling up
e.stopImmediatePropagation()Stops bubbling AND other listeners on same element
e.clientX / e.clientYMouse position relative to viewport
e.pageX / e.pageYMouse position relative to entire page
e.offsetX / e.offsetYMouse position relative to the target element

e.target vs e.currentTarget

e.target = the element you actually clicked. e.currentTarget = the element the listener is attached to. They differ during event bubbling.

preventDefault()

Stops the browser's default behavior for an event.

JavaScript preventDefault 
// Prevent a link from navigating:
link.addEventListener('click', (e) => {
    e.preventDefault();
    console.log('Link click blocked!');
});

// Prevent form submission / page reload:
form.addEventListener('submit', (e) => {
    e.preventDefault();
});

// Prevent right-click context menu:
element.addEventListener('contextmenu', (e) => {
    e.preventDefault();
});

Event Bubbling & Capturing

Event Propagation Phases

When an event occurs, it goes through three phases:

1 Capturing Phase

Event travels DOWN from window → document → ... → target's parent

→
2 Target Phase

Event reaches the actual target element

→
3 Bubbling Phase

Event travels UP from target's parent → ... → document → window

What is Bubbling?

By default, events bubble up. If you click a <button> inside a <div>, the click event fires on the button first, then the div, then body, then html, then document.

JavaScript bubbling 
// Both handlers fire when inner button is clicked!
outer.addEventListener('click', () => console.log('Outer clicked'));
inner.addEventListener('click', () => console.log('Inner clicked'));
// Output: "Inner clicked" → "Outer clicked"

// To STOP bubbling:
inner.addEventListener('click', (e) => {
    e.stopPropagation();   // Outer handler won't fire
    console.log('Inner clicked — stopped here');
});

Capturing Phase

To listen during the capturing phase (top-down), pass { capture: true } as the third argument.

JavaScript capture 
// This fires BEFORE the target's own handler
parent.addEventListener('click', () => {
    console.log('Parent — Capture phase');
}, { capture: true });  // or just: }, true);

// Or shorthand with boolean:
parent.addEventListener('click', handler, true);

Event Delegation

What is Event Delegation?

Instead of attaching a listener to every child element, attach ONE listener to a parent element and use e.target to determine which child was clicked. This is efficient and works for dynamically added elements.

JavaScript delegation 
// Bad: Adding listener to every item
document.querySelectorAll('li').forEach(li => {
    li.addEventListener('click', () => {
        console.log(li.textContent);
    });
});

// Good: One listener on the parent (delegation)
document.querySelector('ul').addEventListener('click', (e) => {
    if (e.target.tagName === 'LI') {
        console.log(e.target.textContent);
    }
    // Also works with classes:
    if (e.target.classList.contains('item')) {
        console.log('Item clicked:', e.target.textContent);
    }
    // Or closest() for nested elements:
    const item = e.target.closest('.item');
    if (item) {
        console.log('Closest item:', item.textContent);
    }
});

DOMContentLoaded vs load

JavaScript load events

document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM is ready!');

Custom Events

Creating Custom Events

You can create and dispatch your own custom events using CustomEvent. This is great for building loosely-coupled components.

JavaScript CustomEvent 
// 1. Create a custom event
const myEvent = new CustomEvent('userLoggedIn', {
    detail: {               // pass any data in 'detail'
        username: 'JohnDoe',
        timestamp: Date.now()
    },
    bubbles: true,          // should it bubble?
    cancelable: true        // can it be cancelled?
});

// 2. Listen for it
document.addEventListener('userLoggedIn', (e) => {
    console.log(`${e.detail.username} logged in!`);
    console.log(`At: ${e.detail.timestamp}`);
});

// 3. Dispatch it
document.dispatchEvent(myEvent);

Pass { once: true } to automatically remove the listener after it fires once.
